### 줄일 수 있는 부분

1. 템플릿 구조 재탐색 최소화
    index.html, answer-template.html, 렌더 스크립트는 한 번 파악된 고정 계약입니다. 새 이미지가 들어와도 매번 긴 sed/rg로 템플
    릿 내부를 읽을 필요는 적습니다.
    필요한 확인은 AGENTS.md, 렌더 스크립트 CLI 옵션, 출력 검증 정도면 충분합니다.

2. 터미널에 긴 JSON 출력하지 않기
    새 이미지마다 JSON은 새로 만들어야 하지만, 만든 JSON을 터미널에 길게 출력할 필요는 없습니다. 검증은 개수와 구조만 출력하면
    됩니다:

    python3 -c "import json; d=json.load(open('student.json')); print(len(d[0]['sourceSentences'])); print([s['title'] for s in
    d[0]['steps']])"

    JSON 파일 생성 자체는 필요하지만, 대화 컨텍스트에 전체 내용을 싣지 않는 게 핵심입니다.

3. 이미지 판독 보고를 짧게 하기
    판독 가능 여부만 말하고, 원문 전체를 중간 보고에 반복하지 않습니다.
    예: “24문장 판독 가능. 손글씨/표시는 제외하고 인쇄 원문만 사용합니다.”
    불확실한 경우에만 문제 구간을 짧게 인용해서 중단합니다.

4. JSON 생성은 스크립트화
    가장 큰 절감 여지는 수동으로 10단계 JSON을 매번 길게 구성하지 않는 것입니다.
    새 이미지에서 추출한 sourceSentences와 최소 메타데이터만 넣으면 10단계 학생용/답지 JSON을 생성하는 별도 스크립트를 두면, 대
    화 토큰과 실수 가능성이 같이 줄어듭니다.

5. 검수 출력 줄이기
    pdfinfo, placeholder 검사, 문장 수, 단계명만 출력하면 됩니다. find 전체 출력이나 큰 로그는 생략합니다.
    최종 보고 파일 목록도 기본 산출물 중심으로 제한합니다.

6. 시각 검수는 대표 페이지만
    AGENTS.md 요구사항은 유지하되, 도구로 확인할 이미지는 최소화합니다:

    - 문제 1쪽
    - part 전환 첫 페이지
    - 중간 단계 1쪽
    - 문제 마지막 쪽
    - 해설 정답 표시가 있는 1쪽
    
    검수 결과는 한 문장으로 보고합니다.

### 새 이미지 기준 저토큰 실행 흐름

1. AGENTS.md 확인
2. 입력 이미지 판독
3. 불확실하면 즉시 중단 보고
4. 판독 가능하면 sourceSentences만 내부 확정
5. 생성 스크립트로 student.json, answer.json 작성
6. 렌더 스크립트 실행
7. 짧은 자동 검증: placeholder, 문장 수, 단계명, PDF 페이지 수
8. 대표 PNG 4~5장 시각 확인
9. 최종 보고: 출력 폴더, 기본 파일 목록, 총 문장 수

핵심은 “새 이미지 판독은 피할 수 없지만, 템플릿 탐색·JSON 전문 노출·검수 로그 노출은 줄일 수 있다”입니다. 가장 효과적인 개선은
10단계 JSON 생성기를 별도 스크립트로 고정해 두는 것입니다.
