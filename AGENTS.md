# AGENTS.md

이 폴더는 영어 지문을 현재 확정된 워크북 디자인으로 변환하는 작업공간이다.

## 최우선 기준

- 학생용 워크북 디자인 템플릿은 `index.html`이다.
- 답지/해설 디자인 템플릿은 `answer-template.html`이다.
- `AGENTS(34).md`는 이전 폐쇄형 7단계 작업 기준이므로 현재 워크북 생성에는 사용하지 않는다.
- 새 HTML을 만들 때 별도 CSS, 별도 표 구조, 별도 카드 구조를 만들지 않는다. 데이터만 바꾸고 레이아웃은 `index.html`과 `answer-template.html`이 강제한다.
- 예전 방식의 B1/B2/B3/B4, `흐름` 열, `뜻 작성` 열, 7단계 폐쇄형 레이아웃이 나오면 실패다.

## 입력 처리

- 원본 이미지·PDF·스캔 자료가 흐릿해서 영어 원문, 선택지, 문장부호, 빈칸 위치, 지문 번호를 확정할 수 없으면 추측하지 않고 선명한 자료를 다시 요청한다.
- 판독 가능하면 바로 원문 전체 문장을 추출하고 `sourceSentences`에 저장한다.
- 문장 단위 연습 유형은 원문 전체 문장 수와 같은 문항 수로 만든다. 대표 문장만 뽑지 않는다.

## 고정 단계

학생용과 답지 모두 아래 10단계 흐름을 유지한다.

1. `지문 연습하기`
2. `빈칸 완성하기 (우리말)`
3. `빈칸 완성하기 (영문)`
4. `해석 연습하기`
5. `동사형 연습하기`
6. `어법·어휘 고르기`
7. `어색한 곳 찾기`
8. `순서 배열하기`
9. `문단 배열하기`
10. `영작 연습하기`

긴 단계는 한 페이지에 억지로 넣지 말고 같은 단계 번호와 제목을 유지해 `part`로 나누어 다음 페이지로 넘긴다.

## 문항 번호·분량 기준

- `part`는 실제 원문 문장 번호 범위를 표시한다. 예: `1-14`, `15-24`.
- 다음 페이지로 넘어가도 문항 번호를 다시 1부터 시작하지 않는다. `item.no`와 `unit.no`에 원문 기준 번호를 넣어 이어지게 한다.
- 문장 단위 단계의 총 문항 수는 원문 전체 문장 수와 같아야 한다.
- `지문 연습하기`처럼 의미 단위·미니 해석이 함께 들어가는 단계는 한 페이지에 약 12문항을 우선 배치한다. 문장이 짧다는 확신이 없으면 13문항 이상 넣지 않는다.
- 빈칸 완성, 동사형, 어법·어휘처럼 답안 작성 공간이 작거나 없는 단계는 한 페이지에 약 14문항을 우선 배치한다.
- 해석 연습처럼 작성줄이 큰 단계는 한 페이지에 약 10문항을 우선 배치한다.
- 순서 배열하기는 제시어·배열칸·작성줄이 함께 들어가므로 한 페이지에 약 6문항을 우선 배치한다.
- 영작 연습하기는 우리말·제시어·작성줄이 함께 들어가므로 한 페이지에 약 8문항을 우선 배치한다.
- 실제 PDF에서 글자 잘림·겹침·하단 침범이 생기면 14문항 기준보다 시각 검수를 우선하여 문항 수를 줄인다.
- 한 페이지 하단을 침범한 내용이 다음 페이지 상단과 겹쳐 보이면 실패다. 같은 단계의 `part` 문항 수를 줄여 다시 렌더링한다.

## 배치 기준

- 영문과 우리말이 함께 나오는 문장 단위 문제는 영문 위, 우리말 아래의 세로 배치를 기본으로 한다.
- 임의로 좌우 2열 배치로 바꾸지 않는다. `layout: "reading"`은 사용자가 좌우 배치를 명시적으로 요구한 경우에만 쓴다.
- 같은 단계가 `part`로 나뉘어도 단계 번호, 단계 제목, 문항 구조, 답지 구조는 동일하게 유지한다.

## 답지 기준

- 답지는 학생용 워크북과 같은 단계·박스·문항 구조를 유지한다.
- 정답만 포인트 빨강 `#D71920`으로 표시한다.
- 문제 본문 전체나 해석 본문 전체가 빨간색이 되면 실패다.
- 기존 `answer-fill` 표시가 답지에서 자동으로 빨간색 처리되면 실패다. 정답 필드에서 새로 렌더링한 값만 빨간색이어야 한다.

## 출력

- 결과는 `outputs/<작업폴더명>/`에 저장한다.
- 기본 파일명은 다음 4개다.
  - `문제.html`
  - `문제.pdf`
  - `해설.html`
  - `해설.pdf`
- 같은 폴더가 있으면 덮어쓰지 않고 `_v2`, `_v3`처럼 새 폴더를 만든다.
- HTML/PDF 생성은 `scripts/render_workbook_from_template.py`를 사용한다.

## 자동 검토

- `index.html`을 직접 열었을 때 현재 확정된 10단계 워크북 미리보기가 보여야 한다.
- `answer-template.html`을 직접 열었을 때 현재 확정된 10단계 답지 미리보기가 보여야 한다.
- 최종 출력 HTML에는 `__DOCUMENT_TITLE__`, `__SCREEN_TITLE__`, `__WORKSHEETS__`가 남아 있으면 실패다.
- 최종 출력 HTML에는 템플릿 미리보기용 과거 데이터가 남아 있으면 실패다. 렌더 스크립트는 출력 시 `previewWorksheets`를 비운다.
- 최종 출력 HTML을 브라우저에서 렌더링했을 때 `페이지 넘침 검토 오류`가 나오면 실패다. `scripts/render_workbook_from_template.py`는 이 오류를 감지하면 PDF 생성 전에 중단해야 한다.
- `blank` 또는 `제시어`가 있는 문항은 제시어 개수와 빈칸·배열칸·작성 대상 개수가 같아야 한다.
- `영작 연습하기` 학생용에는 영어 골격 문장이나 빈칸 영어 문장을 제공하지 않는다. 우리말, 제시어, 답안 작성줄만 둔다.
- PDF 생성 후 첫 페이지, `part`가 넘어가는 다음 페이지, 중간 단계, 마지막 페이지를 PNG로 렌더링해 번호 이어짐, 세로 배치, 하단 잘림, 답지 빨간 정답 표시를 눈으로 확인한다.
