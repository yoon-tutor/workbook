# 10단계 영어 워크북 생성 작업공간

이 폴더는 영어 지문을 정해진 디자인의 학생용 워크북과 답지로 변환하는 작업공간입니다. 핵심은 새 디자인을 매번 만드는 것이 아니라, 이미 확정된 HTML 템플릿에 지문 데이터만 넣어 같은 형식의 결과물을 안정적으로 만드는 것입니다.

## 한눈에 보기

- 학생용 워크북 템플릿: `index.html`
- 답지/해설 템플릿: `answer-template.html`
- HTML/PDF 생성 스크립트: `scripts/render_workbook_from_template.py`
- 입력 데이터 예시: `tmp/pungdong_external/*.json`
- 최종 결과물 위치: `outputs/<작업이름>/`

결과물은 기본적으로 아래 4개 파일로 만들어집니다.

- `문제.html`
- `문제.pdf`
- `해설.html`
- `해설.pdf`

## 이 프로젝트의 원칙

이 프로젝트에서 가장 중요한 것은 레이아웃을 고정하는 것입니다. 워크북 디자인은 `index.html`, 답지 디자인은 `answer-template.html`이 담당합니다. 새 작업을 할 때는 CSS나 카드 구조를 새로 만들지 않고, JSON 데이터의 내용만 바꿔서 같은 레이아웃에 넣습니다.

예전 폐쇄형 7단계 형식은 현재 기준이 아닙니다. `AGENTS(34).md`는 과거 작업 기준이므로 새 워크북을 만들 때 사용하지 않습니다.

## 고정된 10단계 구성

학생용과 답지는 항상 같은 10단계 흐름을 사용합니다.

1. 지문 연습하기
2. 빈칸 완성하기 (우리말)
3. 빈칸 완성하기 (영문)
4. 해석 연습하기
5. 동사형 연습하기
6. 어법·어휘 고르기
7. 어색한 곳 찾기
8. 순서 배열하기
9. 문단 배열하기
10. 영작 연습하기

지문이 길어서 한 페이지에 다 들어가지 않으면 같은 단계 번호와 제목을 유지한 채 `part`를 나누어 다음 페이지로 넘깁니다. 예를 들어 1단계가 `part: "1-12"`와 `part: "13-24"`처럼 나뉠 수 있습니다.

## 데이터가 하는 일

템플릿은 모양을 담당하고, JSON은 내용을 담당합니다. JSON에는 워크북 제목, 학년, 주제, 단계별 문항, 정답 등이 들어갑니다.

기본 구조는 대략 아래처럼 생각하면 됩니다.

```json
[
  {
    "no": "Chocolate History",
    "kicker": "풍동중3 외부지문",
    "grade": "중3",
    "topic": "지문 요약",
    "steps": [
      {
        "no": "1",
        "title": "지문 연습하기",
        "part": "1-12",
        "instruction": "영문과 해석을 읽고 문장의 의미를 이해해 보세요.",
        "units": []
      }
    ]
  }
]
```

문장 단위 단계에서는 원문 전체 문장 수와 문항 수가 같아야 합니다. 대표 문장만 골라 넣으면 안 됩니다. 다음 페이지로 넘어가도 문항 번호는 다시 1부터 시작하지 않고 원문 기준 번호를 이어서 사용합니다.

## 학생용과 답지의 차이

학생용 JSON은 학생이 풀 문제를 담습니다. 빈칸, 선택지, 배열칸, 작성줄 등이 들어갑니다.

답지 JSON은 같은 단계와 같은 문항 구조를 유지하면서 정답을 보여줍니다. 정답만 빨간색 `#D71920`으로 표시되어야 하며, 문제 본문 전체나 해석 전체가 빨간색이 되면 안 됩니다.

특히 `answer-fill` 같은 학생용 빈칸 표시가 답지에서 자동으로 빨갛게 보이면 실패입니다. 답지는 정답 필드에서 렌더링한 값만 빨간색이어야 합니다.

## 생성 명령

이 프로젝트에서 `uv`를 사용할 때는 아래 경로의 `uv`를 사용합니다.

```bash
/Users/jmkim/.pyenv/versions/kookmin-feed/bin/uv
```

학생용과 답지를 HTML/PDF로 만들 때는 다음처럼 실행합니다.

```bash
/Users/jmkim/.pyenv/versions/kookmin-feed/bin/uv run scripts/render_workbook_from_template.py \
  --student-json tmp/pungdong_external/02_Chocolate_History_student.json \
  --answer-json tmp/pungdong_external/02_Chocolate_History_answer.json \
  --out-name chocolate_reading \
  --pdf
```

스크립트는 `outputs/chocolate_reading/` 폴더를 만듭니다. 같은 이름의 폴더가 이미 있으면 덮어쓰지 않고 `chocolate_reading_v2`, `chocolate_reading_v3`처럼 새 폴더를 만듭니다.

PDF가 필요 없고 HTML만 확인하고 싶다면 `--pdf`를 빼면 됩니다.

## 파일 구조

```text
.
├── AGENTS.md
├── README.md
├── index.html
├── answer-template.html
├── assets/
│   └── yonjogyo-logo-footer.png
├── inputs/
├── outputs/
├── scripts/
│   └── render_workbook_from_template.py
└── tmp/
```

각 파일과 폴더의 역할은 다음과 같습니다.

- `AGENTS.md`: 이 작업공간에서 반드시 지켜야 하는 운영 규칙입니다.
- `index.html`: 학생용 워크북의 고정 템플릿입니다.
- `answer-template.html`: 답지/해설의 고정 템플릿입니다.
- `scripts/render_workbook_from_template.py`: JSON 데이터를 템플릿에 넣고 HTML/PDF를 생성합니다.
- `assets/`: 워크북에 들어가는 이미지 자산입니다.
- `inputs/`: 원본 자료를 넣는 폴더로 사용할 수 있습니다.
- `tmp/`: 작업 중 만든 JSON, 검수용 이미지 등 임시 산출물이 들어갑니다.
- `outputs/`: 최종 HTML/PDF 결과물이 저장됩니다.

## 검수 기준

결과물을 만든 뒤에는 단순히 파일이 생성됐는지만 보지 말고 실제 페이지가 잘 보이는지 확인해야 합니다.

확인할 내용은 아래와 같습니다.

- 최종 HTML에 `__DOCUMENT_TITLE__`, `__SCREEN_TITLE__`, `__WORKSHEETS__` 같은 템플릿 토큰이 남아 있지 않은가
- 템플릿 미리보기용 과거 데이터가 결과물에 남아 있지 않은가
- 문항 번호가 페이지를 넘어가도 자연스럽게 이어지는가
- 영문과 우리말이 기본적으로 세로 배치로 보이는가
- 페이지 하단에서 글자가 잘리거나 다음 페이지와 겹치지 않는가
- `bank` 또는 `제시어`가 있는 문항에서 제시어 수와 빈칸/배열칸/작성 대상 수가 맞는가
- 영작 연습하기 학생용에 영어 골격 문장이나 빈칸 영어 문장이 들어가지 않았는가
- 답지에서 정답만 빨간색으로 보이는가

PDF까지 만든 경우에는 첫 페이지, `part`가 넘어가는 페이지, 중간 단계, 마지막 페이지를 PNG로 렌더링해 눈으로 확인하는 것이 좋습니다.

## 자주 나는 실패

아래 상태가 나오면 현재 기준에서는 실패입니다.

- 예전 7단계 레이아웃이 섞임
- B1/B2/B3/B4, `흐름` 열, `뜻 작성` 열 같은 과거 구조가 나타남
- 문항 번호가 새 페이지에서 다시 1번으로 시작함
- 원문 전체 문장 수보다 문장 단위 문항 수가 적음
- 한 페이지에 너무 많은 문항을 넣어 하단이 잘림
- 답지에서 정답뿐 아니라 본문 전체가 빨간색이 됨
- 영작 연습하기 학생용에 영어 정답 형태가 미리 보임

## 작업 흐름

새 워크북을 만들 때는 보통 아래 순서로 진행합니다.

1. 원본 자료에서 영어 원문, 선택지, 문장부호, 빈칸 위치, 지문 번호를 정확히 판독합니다.
2. 원문 전체 문장을 문장 단위로 나눕니다.
3. 학생용 JSON과 답지 JSON을 만듭니다.
4. 10단계가 모두 들어갔는지 확인합니다.
5. `scripts/render_workbook_from_template.py`로 HTML/PDF를 생성합니다.
6. 브라우저와 PDF 이미지로 넘침, 번호, 정답 색상, 배치를 검수합니다.
7. 문제가 있으면 JSON의 `part` 분량이나 문항 내용을 조정한 뒤 다시 렌더링합니다.

원본 자료가 흐릿해서 확정할 수 없는 부분이 있으면 추측하지 않습니다. 영어 원문, 선택지, 문장부호, 빈칸 위치, 지문 번호가 불확실하면 선명한 자료를 다시 받아야 합니다.
